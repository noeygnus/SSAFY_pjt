# pjt05 금융 자산 정보 토론 게시판

## 구현 기능

- 금융 자산 목록과 자산별 토론 게시판을 제공합니다.
- 커스텀 유저 모델을 사용해 `nickname`, `interest_stocks`, `profile_image`를 저장합니다.
- 회원가입, 로그인, 로그아웃, 비밀번호 변경 기능을 제공합니다.
- 회원가입 성공 시 자동 로그인 후 메인 페이지로 이동합니다.
- 게시글 작성은 로그인 사용자만 가능하며, 작성자는 로그인 사용자의 `username`으로 자동 저장됩니다.
- 게시글 수정과 삭제는 작성자 본인만 가능합니다.
- 게시글 상세 페이지에서 댓글 목록, 댓글 작성, 댓글 삭제를 지원합니다.
- 댓글 작성과 삭제도 로그인 및 작성자 권한을 기준으로 제한합니다.
- 프로필 페이지에서 프로필 이미지, 닉네임, 관심 종목, 내가 작성한 게시글, 내가 작성한 댓글을 확인할 수 있습니다.
- LLM API를 활용해 내 게시글과 댓글 기반 투자 성향 분석 결과를 제공합니다.

## 주요 구현 흐름

1. `accounts` 앱을 추가하고 `AbstractUser` 기반 커스텀 유저 모델을 생성했습니다.
2. `settings.AUTH_USER_MODEL`을 설정해 Django 인증 시스템이 커스텀 유저를 사용하도록 연결했습니다.
3. 회원가입 폼에서 `data/assets.json`의 금융 자산 목록을 관심 종목 체크박스로 표시하고, 선택값은 쉼표로 구분해 저장했습니다.
4. 기존 `Post.author` 문자열 필드를 유지하면서 로그인 사용자의 `username`을 자동 저장하도록 변경했습니다.
5. 게시글과 댓글의 수정/삭제 권한은 저장된 작성자 문자열과 현재 로그인 사용자의 `username`을 비교해 판단했습니다.
6. `MEDIA_ROOT`, `MEDIA_URL`을 설정해 개발 환경에서 프로필 이미지를 업로드하고 확인할 수 있도록 구성했습니다.
7. `MODE` 환경변수에 따라 OpenAI 또는 Upstage API를 선택해 투자 성향 분석을 실행하도록 구성했습니다.

## 실행 방법

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

## LLM 설정

`.env` 예시:

```text
MODE=OPENAI
OPENAI_API_KEY=...
OPENAI_MODEL=gpt-5-nano
UPSTAGE_API_KEY=...
UPSTAGE_MODEL=solar-mini
```

`MODE=UPSTAGE`로 변경하면 Upstage OpenAI-compatible API를 사용합니다.

## 확인한 항목

```bash
python manage.py makemigrations --check --dry-run
python manage.py migrate
python manage.py check
python manage.py test
```

추가로 Django 테스트 클라이언트로 다음 흐름을 확인했습니다.

- 메인 페이지 렌더링
- 비로그인 사용자의 글쓰기 접근 시 로그인 페이지로 리다이렉트
- 회원가입 성공 및 관심 종목 저장
- 게시글 작성 시 작성자 자동 저장
- 게시글 상세 페이지 렌더링
- 댓글 작성
- 프로필 페이지 렌더링
- 다른 사용자의 게시글 삭제 차단
- 게시글과 댓글이 부족할 때 투자 성향 분석 안내 메시지 표시

## 학습 내용

- Django에서 커스텀 유저 모델은 프로젝트 초기에 설정하는 것이 가장 안전하다는 점을 확인했습니다.
- `login_required`와 서버 측 권한 검사를 함께 사용해야 화면 버튼을 숨기는 것만으로는 막을 수 없는 직접 요청을 차단할 수 있습니다.
- 명세서가 `Post.author`를 문자열로 유지하도록 요구했기 때문에, 외래 키 대신 `username` 비교 방식으로 권한을 구현했습니다.
- 파일 업로드를 위해 모델 필드뿐 아니라 `MEDIA_ROOT`, `MEDIA_URL`, 개발 서버의 media URL 연결이 함께 필요하다는 점을 정리했습니다.
- LLM 호출은 네트워크와 비용이 발생하므로 글 작성 시 자동 실행하지 않고, 사용자가 분석 페이지에 진입했을 때만 실행하도록 분리했습니다.

## 어려웠던 점

- 기존 프로젝트가 작성자 직접 입력 방식으로 되어 있어 로그인 사용자 기반 권한 흐름으로 바꾸는 과정에서 뷰와 템플릿을 함께 조정해야 했습니다.
- 관심 종목은 DB 모델이 아니라 JSON 파일을 기준으로 표시해야 했기 때문에, 폼 초기화 시점에 선택지를 동적으로 구성했습니다.
- 커스텀 유저 모델 도입 후 기존 인증 URL, 프로필 페이지, 게시글 권한이 서로 맞물리도록 URL 네임스페이스와 리다이렉트 경로를 정리하는 데 주의했습니다.
- LLM 분석은 활동 내용이 부족할 때 결과 품질이 떨어질 수 있어, 최소 텍스트 길이 검사를 먼저 수행하도록 했습니다.
